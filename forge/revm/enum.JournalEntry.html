Expand description

Journal entries that are used to track changes to the state and are used to revert it.

Variants§

§

AccountWarmed

Used to mark account that is warm inside EVM in regard to EIP-2929 AccessList. Action: We will add Account to state. Revert: we will remove account from state.

Fields

§address: Address

Address of warmed account.

§

AccountDestroyed

Mark account to be destroyed and journal balance to be reverted Action: Mark account and transfer the balance Revert: Unmark the account and transfer balance back

Fields

§had_balance: Uint<256, 4>

Balance of account got transferred to target.

§address: Address

Address of account to be destroyed.

§target: Address

Address of account that received the balance.

§destroyed_status: SelfdestructionRevertStatus

Status of selfdestruction revert.

§

AccountTouched

Loading account does not mean that account will need to be added to MerkleTree (touched). Only when account is called (to execute contract or transfer balance) only then account is made touched. Action: Mark account touched Revert: Unmark account touched

Fields

§address: Address

Address of account that is touched.

§

BalanceChange

Balance changed Action: Balance changed Revert: Revert to previous balance

Fields

§old_balance: Uint<256, 4>

New balance of account.

§address: Address

Address of account that had its balance changed.

§

BalanceTransfer

Transfer balance between two accounts Action: Transfer balance Revert: Transfer balance back

Fields

§balance: Uint<256, 4>

Balance that is transferred.

§from: Address

Address of account that sent the balance.

§to: Address

Address of account that received the balance.

§

NonceChange

Increment nonce Action: Increment nonce by one Revert: Decrement nonce by one

Fields

§address: Address

Address of account that had its nonce changed. Nonce is incremented by one.

§

AccountCreated

Create account: Actions: Mark account as created Revert: Unmark account as created and reset nonce to zero.

Fields

§address: Address

Address of account that is created. On revert, this account will be set to empty.

§is_created_globally: bool

If account is created globally for first time.

§

StorageChanged

Entry used to track storage changes Action: Storage change Revert: Revert to previous value

Fields

§key: Uint<256, 4>

Key of storage slot that is changed.

§had_value: Uint<256, 4>

Previous value of storage slot.

§address: Address

Address of account that had its storage changed.

§

StorageWarmed

Entry used to track storage warming introduced by EIP-2929. Action: Storage warmed Revert: Revert to cold state

Fields

§key: Uint<256, 4>

Key of storage slot that is warmed.

§address: Address

Address of account that had its storage warmed. By SLOAD or SSTORE opcode.

§

TransientStorageChange

It is used to track an EIP-1153 transient storage change. Action: Transient storage changed. Revert: Revert to previous value.

Fields

§key: Uint<256, 4>

Key of transient storage slot that is changed.

§had_value: Uint<256, 4>

Previous value of transient storage slot.

§address: Address

Address of account that had its transient storage changed.

§

CodeChange

Code changed Action: Account code changed Revert: Revert to previous bytecode.

Fields

§address: Address

Address of account that had its code changed.
